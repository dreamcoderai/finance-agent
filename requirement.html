It's a substantial portfolio project that demonstrates practical AI engineering, data integration, and financial domain knowledge. Step 1: Define the MVP (Minimum Viable Product) Let's build the first version with one clear goal: "Answer questions about publicly traded companies using real financial data." Example questions it should handle: "What is Apple's market cap?" "Compare Microsoft and Google." "Show Tesla's revenue growth." "Summarize NVIDIA." "What are Amazon's financial ratios?" We won't tackle news, SEC filings, or portfolio analysis yet. Step 4: Build tools first Instead of asking the AI to know financial data from memory, we'll give it tools to retrieve live information. For example: get_company_info("AAPL") get_stock_price("MSFT") get_financials("NVDA") get_balance_sheet("TSLA") get_income_statement("GOOG") These functions are deterministic and testable. The AI's role is to decide which tool to use and how to explain the results. Step 5: Add the AI agent Once the tools work, we'll connect them to the model using function/tool calling. The flow will look like: User │ ▼ LLM │ ├── get_company_info() ├── get_stock_price() ├── get_financials() ▼ Tool Results ▼ LLM ▼ Final Answer This separation keeps the system reliable and easier to debug. Step 6: Test with real questions We'll verify the agent can answer questions like: "What is Apple's market cap?" "Compare Microsoft and NVIDIA." "Show Tesla's latest revenue." "What sector is JPMorgan in?" Only after this works well will we add news, SEC filings, charts, and portfolio analysis.
