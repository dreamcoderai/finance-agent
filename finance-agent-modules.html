Warehouse setup

warehouse/

One-time Databricks DDL that builds the catalog, schemas, and every table + view. Run with python -m warehouse.<name>. All idempotent (CREATE ... IF NOT EXISTS).

create_catalog.pycreate_catalog()

Creates the finance_catalog Unity Catalog namespace that everything else lives under.

create_schemas.pycreate_schemas()

Creates the schemas: bronze, silver, gold (plus reference, analytics).

create_bronze_tables.pycreate_bronze_tables()

Creates bronze.raw_api_data — the single raw landing table.

columnsid (identity)sourceendpointsymbolingest_timepayloadpayload_hashprocessing_status
payload_hash drives change-based dedup; the table enables delta.feature.allowColumnDefaults.
create_silver_tables.pycreate_silver_tables()

Creates the 10 normalized Silver tables.

symbolscompany_profilestock_pricehistorical_pricesincome_statementbalance_sheetcash_flowquarterly_*
create_gold_tables.pycreate_gold_tables()

Creates Gold mirror tables via CREATE TABLE gold.X AS SELECT * FROM silver.X WHERE 1=0 — so Gold's schema always matches Silver exactly.

Note the one rename: Gold's stock_prices mirrors Silver's historical_prices.

Creates analytics views over the Gold tables.

02

Ingestion

ingestion/

Fetches from the sources and writes raw payloads into Bronze. Resumable, throttled, batched — designed to survive an ~84,000-call job.

ingestion_manager.pyIngestionManager

The pipeline entry point. run(limit) loads SEC symbols, then delegates all Yahoo endpoints to the batch engine.

methodsrun()ingest_sec_symbols()ingest_yahoo()
sec_ingestion.pySECIngestion

Loads the SEC ticker master into silver.symbols (full refresh, batched 500 rows/insert).

methodsingest_symbols()
batch_ingestion.pyBatchIngestor

The all-endpoints engine. Defines the 9 Yahoo endpoints, then for each: skip already-ingested symbols (resume), fetch concurrently with retry/backoff, and bulk-write to Bronze.

methodsrun(limit)run_endpoint()_process_chunk()_fetch_one()_write_batch()endpoints()
Resumable · throttled (4 workers + delay + backoff) · batched. Run one instance at a time.
bronze_loader.pyBronzeLoader

The write layer to Bronze. Serializes payloads (dict → JSON, DataFrame → reset_index().to_json), hashes them, and inserts.

methodsserialize()hash_payload()get_ingested_symbols()insert_batch()load()
hash_payload = SHA-256 for change-dedup · get_ingested_symbols powers resume · insert_batch is size-aware (flushes at 900 KB / 500 rows to stay under Databricks' 1 MB param limit).
03

ETL

etl/

Refines the data tier to tier. Both are full-refresh (DELETE + batched insert) — run one at a time.

bronze_to_silver.pyBronzeToSilverETL

Parses Bronze JSON into the 9 typed Silver tables. Takes the latest snapshot per symbol (via ROW_NUMBER()) and melts financial statements into long format (year · metric · value).

methodsrun()_latest_bronze()_refresh_table()load_company_profiles()load_historical_prices()load_income_statement()…
Inserts use explicit CAST(? AS type) per column to avoid multi-row inline-table type errors.
silver_to_gold.pySilverToGoldETL

Publishes Gold by copying each Silver table 1:1 (INSERT INTO gold SELECT * FROM silver). Gold is the only layer the app reads.

methodsrun()refresh_table()load_company_profile()load_stock_prices()…

Services

services/

The data-access layer. Connectors, the two backends (Databricks & Yahoo), and the provider that unifies them with source tracking.

databricks_connection.pyDatabricksConnection

Singleton SQL connection — sql.connect(server_hostname, http_path, access_token) from env vars. Exposed as databricks_connection.

methodsget_connection()
databricks_service.pyDatabricksService

Reads the Gold tables via SQL, one method per data type.

methodsget_company_info()get_stock_price()get_historical_prices()get_income_statement()get_balance_sheet()get_cash_flow()get_quarterly_*()
market_data.pyMarketDataService

The yfinance wrapper — the live Yahoo backend. Same method surface as DatabricksService, so they're interchangeable.

methodsget_company_info()get_stock_price()get_historical_prices()get_income_statement()…
financial_data_provider.pyFinancialDataProvider

The unifier. AUTO mode tries Databricks first, falls back to Yahoo when Gold has no row — and records which backend served each call. Shared singleton used by all tools.

methods_fetch()set_source()reset_tracking()sources_summary()get_* (all data types)
Modes: AUTO (Databricks → Yahoo) · DATABRICKS · YAHOO. Provenance surfaces as the source badge in the UI.
symbol_service.pySymbolService

Reads the symbol master from silver.symbols — the source of the ingestion work-list.

methodsget_active_symbols()get_symbol()get_all_symbols()
sec_service.pySECService

Fetches SEC's company_tickers.json and returns a DataFrame of cik · symbol · company_name.

methodsget_company_tickers()

06

Agent & app

agents/ · streamlit_app.py

The reasoning layer and the interface a user actually touches.

agents/finance_agent.pyFinanceAgent

Wraps GPT-5 (MODEL = "gpt-5") with the 9 tools as function schemas. ask() runs an agentic loop — call tools, read results, decide, repeat (up to 6 rounds) — then answers with real figures.
